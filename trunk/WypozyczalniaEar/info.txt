kroki niezbedne do odpalenia aplikacji:

1) utworzenie schematu (uzytkownika) oraclowego 'wyp' o hasle 'wyp'
nalezy pamietac o nadaniu prawa create session oraz resource

najlatwiej osiagnac to poprzez database control: wybrac sobie schemat 'scott' 
(po zalogowaniu, administration -> users) i dalej poprzez opcje 'create like'

testem na prawidlowa konfiguracje oracla jest:

sqlplus wyp/wyp
create table tabelka(id int);
insert into table values (2);
select * from tabelka;

2) konfiguracja glassfish (z cmd)

potrzebne sa dwie komendy, u mnie sid oracla to 'master', oczywiscie trzeba podmienic na swoja
poza tym w katalogu domowych glassfisha powinien byc plik z haslem (plain text)

asadmin create-jdbc-connection-pool --host localhost --port 4848 --user admin --passwordfile "C:\Program Files\glassfish-v2ur1\passfile" --datasourceclassname oracle.jdbc.pool.OracleDataSource --restype javax.sql.DataSource --property DataSourceName=OracleDataSource:NetworkProtocol=tcp:DatabaseName=master:Password=wyp:URL=jdbc\:oracle\:thin\:@localhost\:1521\:master:User=wyp:ServiceName=master:PortNumber=1521:ServerName=localhost WypozyczalniaPool

asadmin create-jdbc-resource --host localhost --port 4848 --user admin --passwordfile "C:\Program Files\glassfish-v2ur1\passfile" --connectionpoolid WypozyczalniaPool jdbc/WypozyczalniaPoolDS

3) aplikacje uruchamia sie poprzez run na wypozyczalniaear
debuguje zeby bylo ciekawiej zupelnie inaczej: tzn klikamy debug na ..-ejb,
a potem debug na app-client lub WypozyczalniaTester (test case)
